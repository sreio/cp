手动拉取数据功能设计

支持按期号/日期范围拉取，智能冲突检测

API 接口设计

// 1. 检查数据是否存在（预检查）
POST /api/lottery/check-exists
{
  "type": "ssq",
  "issues": ["2024001", "2024002", "2024003"]
}

Response:
{
  "existing": [
    {"issue": "2024001", "source": "official", "draw_date": "2024-01-02"}
  ],
  "missing": ["2024002", "2024003"]
}

// 2. 拉取数据（带冲突处理）
POST /api/lottery/pull
{
  "type": "ssq",
  "mode": "issue",           -- "issue" | "date_range"
  "issues": ["2024001", "2024002"],  -- mode=issue 时使用
  "start_date": "2024-01-01",    -- mode=date_range 时使用
  "end_date": "2024-01-31",
  "force_update": false,        -- 是否强制更新已存在数据
  "source": "third_party"       -- 数据源优先级
}

Response (无冲突):
{
  "success": true,
  "pulled": 2,
  "skipped": 0,
  "updated": 0
}

Response (有冲突，需要确认):
{
  "success": false,
  "conflict": true,
  "message": "发现已存在的数据",
  "existing_data": [
    {
      "issue": "2024001",
      "source": "official",
      "draw_date": "2024-01-02",
      "red_balls": "[01,05,10,15,20,30]",
      "blue_ball": "03"
    }
  ],
  "missing_data": ["2024002"]
}

数据源适配器设计

支持的数据源

官方 API
  • 中国福利彩票 (cwl.gov.cn)
  • 中国体育彩票 (lottery.gov.cn)
  • 实时数据，权威可靠
  • 有频率限制，需缓存
第三方 API
  • 500.com / zhcw.com
  • opencai.net
  • 历史数据丰富
  • 批量拉取效率高

前端交互流程

步骤 1
用户选择拉取方式（按期号/按日期范围）
步骤 2
系统调用 check-exists 预检查
步骤 3
无冲突：直接拉取并显示结果
有冲突：弹窗提示，显示已存在的数据
步骤 4
用户选择：跳过已存在 / 强制更新全部 / 选择性更新
步骤 5
执行拉取，显示进度和结果
